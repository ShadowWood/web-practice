## 项目及git使用说明

  1. fork 本项目，在fork到自己的项目的地址内开发，建议至少一个dev分支，一个feature分支，在dev分支上用来合并，feature分支上用来开发新功能；

  2. 开发完一定功能后或一定版本后，向本项目的分支提pull request，分支命名为`自己的名字-练习项目名称`，如`wushengyu-practice-1`；

  3. 保证自己的分支整洁，多尝试使用rebase整理自己的分支；
